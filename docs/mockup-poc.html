Mockup Compositor · proof of concept

One reusable prop + one design → a product cover, composited in the browser. Nothing is baked to a file: the cover is a recipe (prop + design + placement). This POC proves the on-chain-native approach before it ports into Pole Position.

How this maps on-chain

The composite you see is never stored. On chain you keep only:

  • reused the prop atom — minted once as a .bmc of maps (base + displacement + shading + mask), shared across the whole catalogue.
  • 1× the design — as a watermarked public preview (the clean file stays encrypted).
  • a few hundred bytes of manifest: { prop, design, place }.
One design → many covers (tee, mug, tote) from tiny manifests, not stored images.

Fidelity note: shading (multiply) and cylindrical wrap (mugs, cans, bottles) are both procedural here. Irregular surfaces — a folded tee, a crumpled tote — instead warp through the prop's authored displacement map (an SVG feDisplacementMap pass) that ships inside the real .bmc prop atom. Same manifest, richer prop.
